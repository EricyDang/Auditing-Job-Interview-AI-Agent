# Analysis Prompts

## Target Clarification

如果目标不明确，请根据简历中的专业、技能、项目、实习、作品和限制条件，推断：

- 最适合主投的职位名称。
- 可以尝试但不是主线的职位。
- 暂不建议投递的职位及原因。
- 一句话求职定位。

如果目标过宽，必须收窄到 1 个主方向 + 2 个辅助方向。

## Role Model Builder

根据目标行业、目标职位和 JD，提炼招聘方看重的能力：

- 硬技能
- 工具/软件/平台
- 行业知识
- 项目或工作经验
- 可量化成果
- 沟通协作能力
- 证书或学历要求
- 作品集或案例要求

判断简历中哪些已经覆盖，哪些明显缺失。

## Resume Audit

从以下角度审查简历：

- ATS 关键词筛选。
- 招聘官 10 秒扫描。
- 经历可信度。
- 成果表达。
- 岗位匹配度。

指出最突出的亮点、不清楚/不可信/不够有竞争力的地方，并给出关键词新增、强化、弱化或删除建议。

## Diagnosis

按严重程度诊断零回复原因。每个问题必须包含：

- 问题表现
- 招聘官视角
- 对面试率的影响
- 具体修改建议
- 判断依据：基于材料可判断 / 因缺少信息只能推断

## Rewrite Engine

直接重写：

- 简历顶部个人定位
- Summary
- 技能区
- 1-3 段最重要经历
- 教育经历
- 奖项/证书/作品集描述
- 自我评价

不得新增用户没有提供的事实。没有数据时，用“建议补充：【数据类型】”提醒。
